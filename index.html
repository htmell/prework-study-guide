HTML

What have I learnt?

  • Keeping your code tidy and ordered is important for collaboration and debugging.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • You can divide sections of a webpage by using div or span elements.
  • The elements is the entire section containing tags.
  • Tags are show in code by using angle brackets
  • It is the developers responsibility to build a good UX.
  • Using elements and tags in your code is the first step to creating a good UX on the webpage.
  • Creating a structure and clear framework using HTML elements will help you to build on it with CSS and JS to build the best UX possible.
  • Using notes and comments in your HTML code will be important to mark prgress and provide help later or to collaborators.

Git

What have I learnt?

  • Using the CLI and GitBash is a quick and efficient to navigate your system.
  • GitHub is a useful tool to keep your code in a central location. This way people can track your progress and also collaborate easily.
  • Some useful commands for the CLI are:
    1. ls - for listing the directories.
    2. cd - change directories.
    3. mkdir - this will create a directory in your current location when followed by a name.
    4. git status - this will show which branch you are currently working in.
    5. git checkout main - this will ensure you are in the main branch.
    6. git pull origin main - this will update the main branch if necessary.
    7. git checkout -b branch-name - this will creat a new branch and will switch to it.
  • GitLab will be the location to keep your code saved duruing the Study Process.
  • Connecting your Git to your local system will esnure everything worked on will be able to be saved in a central location.
  • You can encrypt the connectinon between the local system by creating a SSH (Secure Shell).
  • Once you have an SSH code, you can save to your GitHub account to keep the connection.
